Task Descriptor: VERBETC. A2
The first activity of this task of is to build a model based on existing specification documents (Migration goals). The idea is that this “test model” represents the intended behaviour. The purpose is to have a test model to be later used by a test case generator in order to create the test cases for the migrated application. To allow for reusing existing MBT (Model Based Testing) technologies and enable a more efficient test case generation, PIM should be translated into concern-specific and streamlined specification models, which finally act as the actual models that will be used for the test case generation. Moreover, dedicated specification models allow to first apply certain simplifications, to again limit the potentially huge state space to be tested and, thereby, keep the generated test suite more compact and sufficient a priori.
